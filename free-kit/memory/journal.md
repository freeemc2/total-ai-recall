# Journal — Shared Notepad for Multiple Sessions

*Optional. You only need this if you run more than one AI session at a time. If it's just you and one AI, ignore this file until you don't.*

When you run two or more sessions (say, one writing code and one doing research), they can't see each other. Without a shared notepad, they edit the same files, redo each other's work, and step on each other. This file is that notepad. Think of it as the log a work crew leaves on a job site: who's here, who's doing what, and notes for the next shift.

**The habit:** before starting anything big, read this file. Add a line saying what you're about to do. When you finish, mark it done. Leave notes for the other sessions.

---

## Crew — who's out there

*Each session adds itself here once, so everyone knows who exists.*

- *(example: Session A — building the game UI, on this laptop)*

---

## Active — what's in flight right now

*Add a line when you start something big. Remove it when you're done. This stops two sessions from grabbing the same job.*

- *(example: Session A — rewriting the level loader, ETA 20 min, don't touch levels.js)*

---

## Handoffs — notes from one session to another

*Messages that outlive a session. "Session B: I finished the art, it's in /assets, your turn to wire it up."*

- *(nothing yet)*

---

## Log — what got done

*A running history. Newest at the top. Useful for remembering what happened last week.*

- *(example: Mon — Session A — finished the main menu, works)*
